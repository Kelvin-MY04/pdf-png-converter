# Implementation Plan: Fix Faint Lines in Exported PNG Images

**Branch**: `001-fix-faint-lines` | **Date**: 2026-02-26 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-fix-faint-lines/spec.md`

## Summary

Lines in exported PNG images appear faint because PyMuPDF's default anti-aliasing level (8) blends thin stroke pixels with the background, producing semi-opaque grey marks. The fix adds a `RenderingOptions` value object, wires it into `ConversionConfig` and `config.toml`, and applies `graphics_aa_level = 0` (no AA) before each `get_pixmap()` call in `PdfRenderer`. Text AA is kept at level 8 to preserve readable dimension labels. All stroke colors are affected ŌĆö the fix applies to both B/W and color export modes.

### Step 4: Fix `PdfRenderer`

**File**: `src/pdf_png_converter/services/pdf_renderer.py` (MODIFY)

In `render_page()`, apply AA settings before the DPI calculation (which itself calls `get_pixmap()` for probing):

```python
def render_page(self, source_path, page_number, output_path, config):
    self._apply_rendering_options(config.rendering_options)
    with pymupdf.open(...) as doc:
        ...

def _apply_rendering_options(self, options: RenderingOptions) -> None:
    pymupdf.TOOLS.set_graphics_aa_level(options.graphics_aa_level)
    pymupdf.TOOLS.set_text_aa_level(options.text_aa_level)
```

`_apply_rendering_options()` is a private method that encapsulates the PyMuPDF global state mutation, keeping `render_page()` clean and making the AA application independently testable.

---

### Step 5: Update `config.toml`

**File**: `config.toml` (MODIFY)

Add after `[conversion]` block:

```toml
[conversion.rendering]
# Anti-aliasing level for vector lines and paths (0 = none, 8 = maximum).
# Default 0 ensures all strokes render as fully opaque pixel-exact marks.
graphics_aa_level = 0

# Anti-aliasing level for text glyphs (0 = none, 8 = maximum).
# Default 8 preserves smooth rendering of dimension labels and annotations.
text_aa_level = 8
```

## Risk Register

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| `pymupdf.TOOLS.set_graphics_aa_level` not available in older patch versions | Low (requires Ōēź 1.25.0) | `pyproject.toml` already pins `pymupdf>=1.25.0`; method available since 1.18.0 |
| Disabling AA causes text in PDFs to appear jaggy | Low | `text_aa_level` is kept at 8; separate API controls graphics and text independently |
| Fixture `lines.pdf` coordinate-based pixel sampling is fragile | Medium | Sample a range of pixels along the line (not a single pixel); use a threshold (Ōēź 80% of sampled pixels must pass) rather than 100% to allow for sub-pixel boundary effects |
| Global AA state leaks between test runs | Low | Unit tests mock `pymupdf.TOOLS`; integration test runs full render and restores state in teardown |
